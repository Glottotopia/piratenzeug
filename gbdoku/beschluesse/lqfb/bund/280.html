Es ist weithin Konsens, dass HTML-Mails und (größere) Anhänge nicht auf Mailinglisten gehören. Dennoch geraten immer wieder nervige (und potentiell gefährliche) HTML-Formatierungen, Fotos, PDFs etc. auf die Liste. In den meisten Fällen dürfte das aus Versehen oder aus Unkenntnis geschehen.

Es existieren vollwertige technische Lösungen:

1. Die Listenadministratoren können Mailman so einreichten, dass HTML in Nurtext umgewandelt wird ("convert text/html parts to plain text"). 2. Mails ab einer gewissen Größe, z.B. 50 KB, kann die Auslieferung versagt werden ("Maximum length in Kb of a message body"). Der Absender wird automatisch benachrichtigt, falls seine Mail aufgrund ihrer Größe zurückgewiesen wurde.

Die Mailinglisten werden dadurch in ihrer Funktionalität gestärkt. Auch von einem Lerneffekt bei den Nutzern ist auszugehen. Wo Dokumente oder HTML-Formatierung unerlässlich erscheinen, können diese mittels Verlinkung problemlos zugänglich gemacht werden.

Mithin: Die Administratoren sollten beauftragt werden, die beiden genannten Änderungen auf Mailinglisten der Piratenpartei in die Tat umzusetzen.

Anregungen
 
 
 
 
 
HTML bouncen
 
 
 
Keine Anhänge bei Mails (also nicht nur keine größeren)
