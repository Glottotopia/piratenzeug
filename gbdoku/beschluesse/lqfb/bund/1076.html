Letzter Entwurf vom 17.09.2010 um 11:43:37 Uhr · Zeige alle Versionen (2) · Short URL: lfpp.de/i1076

Antrag

Die LiquidFeedback Frontend-API sollte auch über HTTP, also unverschlüsselt, zugänglich sein.

Begründung

Der derzeitige Funktionsumfang der API beinhaltet keinerlei zu verschlüsselnde Daten. Zudem ist die API ja ohne Benutzerkennung zugänglich.

Es gibt gleich 2 Nachteile die gegen eine API-Verbindung über SSL sprechen:

1. Für Tool-Programmierer stellt das Importieren von Zertifikaten eine große Hürde dar, gerade wer in Java programmiert, weiss, dass diese Prozedur für den Endanwender kaum zuzumuten ist.

2. Auf dem Server erzeugt HTTPS deutlich mehr Last, als eine HTTP Verbindung.
