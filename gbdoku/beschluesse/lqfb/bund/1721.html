Idee

Das LQFB-Team implementiert ein System zum einfachen Auffinden geeigneter Delegierter. Dieses soll direkt in die Software eingebunden sein beziehungsweise über eine Schnittstelle direkt mit dieser kommunizieren können.

Mitglieder tragen in einen Fragebogen ihre Ansichten zu bestimmten Themen ein und gewichten diese. Das System sucht dann nach aktiven anderen Mitgliedern, die diese Schema möglichst entsprechen. Diese werden dann dem Suchenden vorgeschlagen und er hat direkt die Möglichkeit, den Suchvorschlag in eine Delegation einzusetzen.

Als Grundlage für diese Fragebögen sollen zwei miteinander verwobene Datenquellen verwendet werden: Zum einen sollen von einer Redaktion regelmäßig Fragen zu aktuell wichtigen Themen eingepflegt werden. Zum Anderen weist diese Redaktion auch den einzelnen Initiativen im LQFB bestimmte Schlagworte zu, die dann der Delegiertenfinder auswerten kann. Über das Abstimmverhalten des Delegierten wird so ebenfalls sein Profil verändert.

Details

Im folgenden Abschnitt wird die Grundidee weiter konkretisiert. Details und zusätzliche Funktionen sind in separaten Initiativen dieses Themas zu finden.

Aus Sicht des Delegationssuchenden

Der Delegationssuchende kann an verschiedenen Stellen des LQFB-Systems auf den Finder zugreifen. Dieser verhält sich dem Thema entsprechend anders. Gesucht werden können folgende Kandidaten:

  • Globaler Delegierter
    • Hier wird ein breites Themenfeld abgefragt. Es wird ein Kandidat gesucht, der in seiner Gesamtheit eine möglichst breite Übereinstimmung bringt.
  • Thematischer Delegierter
    • Es werden Fragen zum gewählten Themenbereich gestellt.
  • Fachdelegierter
    • Hier wird ein Delegierter speziell für ein bestimmtes Thema gesucht.

Das System soll anhand des globalen Delegierten erklärt werden: Der Benutzer sieht zunächst eine Themenwolke. Diese Themen sollen grobe Oberbegriffe darstellen wie »Wirtschaft«, »Verkehrspolitik« oder »Familienpolitik«. Diese Themen kann der Benutzer aus der Wolke ziehen und auf einer Skala graphisch nach Wichtigkeit anordnen. Besonders wichtige Themen erscheinen dabei oben, unwichtige unten. Themen, die in der Wolke verbleiben werden ignoriert.

Im folgenden Schritt werden dem Benutzer ausformulierte Ja/Nein/Egal-Fragen gestellt. Jede Frage kann außerdem über ein noch festzulegendes System gewichtet werden. Pro Seite erscheinen circa zehn Fragen, um die Übersichtlichkeit zu gewähren.

Die Art der Fragen richtet sich nach der vorherigen Themenauswahl. Aus hoch gewichteten Themenbereichen werden tendenziell mehr Fragen eingeblendet als aus unwichtigen. Jedoch ist dafür zu sorgen, dass es zu keiner einseitigen Befragung kommt.

Der Benutzer kann so viele Fragen beantworten wie er möchte. Ein Speichern und Unterbrechen der Befragung soll jederzeit möglich sein. Der Benutzer beendet die Befragung wenn er der Meinung ist, dass das System nun genug über ihn weiß. Hier könnte eventuell über einen Hilfsindikator nachgedacht werden, der dem Benutzer sagt wie genau der Finder ihn einschätzen kann.

Nach Beendigung der Befragung werden sofort geeignete Kandidaten gelistet. Bei der Reihenfolge wird nicht nur die thematische Übereinstimmung gewichtet sondern auch die Aktivität des Kandidaten. So soll ausgeschlossen werden, dass es zu einer Schwämme von Hinterbänkler-Delegierten kommt.

Das System sollte in dieser Situation alternativ auch anbieten statt einer globalen Delegation mehrere thematische Delegationen anzulegen, wenn hier eine höhere Gesamtübereinstimmung möglich wäre.

Aus Sicht des potentiellen Delegierten

Aus Sicht des potentiellen Delegierten ist zunächst nichts gesondertes zu beachten. Diejenigen Einträge, die es selbst in sein Suchprofil eingibt werden ebenfalls bei der Suche mit ausgewertet.

Allerdings wird auch das Abstimmverhalten beschrieben und zu Rate gezogen. Zunächst einmal wird dadurch die Aktivität des Mitglieds bestimmt. Selben Abstimmende werden in der Liste weiter hinten aufgelistet. Außerdem weißt eine Redaktion jedem Thema im LQFB bestimmte Tags und Fragen zu, die dann wiederum im Fragebogen gelistet werden.

Datenschutz

Der Datenschutz spielt natürlich auch bei diesem System eine wichtige Rolle. Die Funktion des Delegiertenfinders muss händisch im Benutzerprofil aktiviert werden. Vorher werden keine Daten erfasst oder gespeichert. Nutzer, die nicht an dem Programm teilnehmen können auch nicht aufgefunden werden.

Außerdem gibt es durch eine zweite Auswahl die Möglichkeit, sich selbst aus dem Suchindex zu streichen. Dann kann man über das System nicht mehr aufgefunden werden, jedoch trotzdem die Dienst des Auffindens geeigneter Delegierter in Anspruch nehmen.

Der Benutzer hat außerdem jederzeit die Möglichkeit einzelne seiner Antworten zu überdenken oder zu löschen. Die Historie der gegebenen Antworten wird über die Profileinstellungen bearbeitet. Ein händisches Eingreifen in die automatisch erfassten Daten des Abstimmverhaltens ist jedoch nicht möglich.

Beim Deaktivieren der Funktion hat der Nutzer die Wahl die Daten auf dem Server zu belassen oder diese dauerhaft und restlos zu löschen.

Umsetzung

Die Umsetzung der Änderung kann über mehrere Weisen erfolgen. Eine Möglichkeit bestünde darin, dass vom Vorstand berufene Freiwillige die Änderung direkt in der Software vornehmen. Alternativ könnte ein derartiges System auch extern umgesetzt werden, welches über die API oder Datenbank-Dumps auf die benötigten Daten zugreift.

Eine Umsetzung durch das Entwicklerteam von LQFB ist derzeit unwahrscheinlich, da diese verlauteten, dass sich keine weiteren Änderungen am System für die Piratenpartei vornehmen würden.

Welches Umsetzungssystem genau bevorzugt wird, soll über eine separate Abstimmung oder einen etwaigen Vorstandsbeschluss entschieden werden.

Begründung und Hintergrund

Gegenwertig wird LQFB nur von einem Bruchteil der Parteimitglieder genutzt. Um allerdings echte repräsentative Mehrheiten zu Stande zu bekommen sollte dieser Wert drastisch erhöht werden. Mitglieder können aus diversen Gründen allerdings nicht oder nur in Teilen an den Entscheidungsprozessen teilhaben:

  • Zu wenig Zeit
  • Fehlende Fachkompetenz
  • Abwesenheit bzw. kein Internetanschluss verfügbar
  • Fehlende Lust, alle Anträge zu erfassen
  • Kein Interesse an bestimmten Themen

Aus diesen und anderen Gründen wurde in LQFB ein Delegationssystem eingeführt. Um dieses jedoch nutzen zu können müssen geeignete Delegierte gefunden werden, die die eigenen Interessen möglichst umfangreich vertreten.

Bisher lautet das Konzept hauptsächlich, dass man persönliche Kontakte herstellt und aus dem Bekanntenkreis eine passende Person findet. Dies birgt jedoch einige Nachteile! Der Größte ist wohl, dass man kaum genügend Personen ausreichend gut kennt um hier eine kompetente Entscheidung treffen zu können. Gerade passivere Mitglieder haben so keine Möglichkeit sich wirklich einzubringen, wenn sie die Abstimmungen nicht alle selbst wahrnehmen wollen.
