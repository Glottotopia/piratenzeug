Letzter Entwurf vom 07.07.2012 um 01:05:48 Uhr · Zeige alle Versionen (2) · Short URL: lfpp.de/i3820

Es kann auf mehrere (n) Nutzer gleichzeitig delegiert werden. Das hierbei auf die jeweiligen Nutzer delegierte Stimmgewicht entspricht 1/n. Wenn also auf zwei Nutzer delegiert wird, wird jeweils das halbe Stimmgewicht übertragen.

Begründung:

Im Allgemeinen wird LiquidFeedback/Liquid Democracy als Symbiose von Elementen direkter und indirekter/parlamentarischer demokratischer Entscheidungsprozesse verstanden. Entscheiden sich Nutzer für das Delegieren, existiert momentan jedoch gegenüber Wahlen in einem parlamentarischen System der Nachteil, dass das gesamte Stimmgewicht an eine einzige Person übertragen wird und so eine starke Tendenz besteht, dass es zu "Superdelegierten" kommt. Bei Parlamenten hingegen wird die Stimme verteilt auf die jeweilige Anzahl an Sitzen, die die Partei bei der Wahl erhalten hat, und so eine zu starke Machtkonzentration auf eine einzelne Person vermieden. Dieser Vorteil ließe sich mit der Ermöglichung von Mehrfachdelegationen nachahmen - die Nutzer wären so z. B. in der Lage mehrere Experten für einen Themenbereich zu "wählen".

Eine Konsequenz wäre, dass es bei den Abstimmungsergebnissen/Unterstützungen zu nicht-ganzzahligen Ergebnissen kommen kann, wenn sich z. B. einer von mehreren Delegierten enthält. Es ist klar, dass das technisch keine triviale Sache, sondern ein sehr elementarer Eingriff wäre (Umstellung von allem auf Brüche), aber hier im Meinungsbild gehts erstmal nur um die Idee.
