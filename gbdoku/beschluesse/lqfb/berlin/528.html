i528: RSS-Feed oder Benachrichtigung für Themenbereich, Themen und anderes
Initiative i528: RSS-Feed oder Benachrichtigung für Themenbereich, Themen und anderes
Diese Initiative wurde am/um 06.03.2010 14:33:18 Uhr zurückgezogen
Letzter Entwurf vom 05.03.2010 um 14:02:01 Uhr

Es sollte dem Liquid Feedback eine Funktion hinzugefügt werden, die das Verfolgen von Themenbereichen und Themen ermöglicht.

Benachrichtigung / Abonnement

  1. Es sollte eine Möglichkeit zum Abonnieren eines Themenbereichs und von Themen eingebaut werden.
  2. Man sollte über eingehende Delegationen benachrichtigt werden.
  3. Die Benachrichtigung zu den einzelnen Punkten sollte globale Einstellungen und spezielle Einstellungen unterstützten
    1. Man sollte einstellen können, ob und wie man Benachrichtigungen grundsätzlich haben möchte.
    2. Man sollte ein spezielles Thema, einen speziellen Themenbereich oder eine spezielle Initiative gesondert abonnieren können.

Verfolgen von Themenbereichen

Was

  1. Alle Themenbereiche bei denen man Mitglied wird sollten standardmäßig abonniert werden.
  2. Es sollte grundsätzliche eine Benachrichtigung für neue Themen erfolgen.

Wie

  1. Für das Verfolgen von Themenbereichen bietet sich ein RSS-Feed (oder vergleichbare Feed-Technik) als Mittel der Benachrichtigung an.
  2. :Es könnte so Problemlos ein Feed für alle Mitglieder erstellt werden, eine Individualisierung des Feeds wäre nicht erforderlich.
  3. Zusätzlich sollte über die Möglichkeit der Benachrichtigung per Mail nachgedacht werden.
  4. :Dies hätte den Vorteil, dass auch Leute, die keine RSS-Feeds nutzen, sich auf dem laufenden halten können.

Verfolgen von Themen

Was

  1. Unterstützt man eine Initiative in einem Thema sollte das standardmäßig Thema abonniert werden.
  2. Es sollte eine Benachrichtigung für neue Initativen zu einem Thema erfolgen.

Wie

  1. Für das Verfolgen von Themen bietet sich, da weniger Änderungen erfolgen, eine Benachrichtigung per Mail an.
  2. :Einer zusätzlichen RSS-Feed Generierung steht, sofern es in einem verhältnismäßigen Aufwand steht, nichts entgegen.

Verfolgen von Initiativen

Was

  1. Ist eine Initiative abonniere sollte man darüber benachrichtigt werden, wenn diese in Abstimmung geht.
  2. Ist die Abstimmung beendet, sollte man darüber benachrichtigt werden, wie das Ergebnis lautet.

Wie

  1. Für das Verfolgen von Initiativen bietet sich, da weniger Änderungen erfolgen, eine Benachrichtigung per Mail an.
  2. :Einer zusätzlichen RSS-Feed Generierung steht, sofern es in einem verhältnismäßigen Aufwand steht, nichts entgegen.

Benachrichtigung über eingehende Delegationen

Was

Im Falle einer neuen eingehenden Delegation zu einem Thema, einem Themenbereich oder einer globalen Delegation sollte eine Benachrichtigung erfolgen.

Wie

Hier bietet sich auf Grund der geringen Anzahl der Änderungen nur eine Benachrichtigung per Mal an.

Anregungen
 
 
Benachrichtigungen sind schon geplant
